Submit a Tool via Pull Request
To maintain quality and accuracy, all tool submissions are reviewed through GitHub Pull Requests.
How to Submit a Tool:
  1. Fork the kubernetes-tools repository
  2. Add your tool information to the scripts.js file in the correct category
  3. Include accurate GitHub star count and other details
  4. Create a pull request with a clear description of the tool
Tool Information Format 
{
    id: [next_available_id],
    name: "Tool Name",
    description: "Brief description of the tool",
    category: "[category-key]",  // Example: cli-tools, monitoring, security
    github_url: "https://github.com/org/repo",
    stars: [github_star_count],
    votes: 0
}
Available Categories:
  • cli-tools - CLI Tools
  • monitoring - Monitoring & Alerting
  • cluster-management - Cluster Management
  • logging - Logging & Tracing
  • troubleshooting - Troubleshooting
  • development - Development Tools
  • cicd - CI/CD Integration
  • security - Security Tools
  • service-mesh - Service Mesh
  • storage - Storage Providers
  • backup - Backup Tools
  • cost - Cost Optimization
  • faas - Function as a Service
  • ai - AI/ML Tools
